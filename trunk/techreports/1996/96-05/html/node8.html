Location

The location dimension indicates where in the system state information is held.

Some systems store all state information on a central server, and require clients to request each value individually. Other systems store state information in a distributed manner, where each client maintains its own state information locally. Still others replicate state information across the clients and servers.

In our experience, the ability to control where state information is located is a major determinant of whether or not the system can be simultaneously expressive and responsive.
