Hackystat Design Notes

Philip Johnson, Cam Moore, Jitender Miglani, Shenyan Zhen
Collaborative Software Development Laboratory
Department of Information and Computer Sciences
University of Hawaii

This page contains pointers to a set of online design note documents that introduce issues related to the development of the Hackystat system for automated collection and analysis of software engineering data.

As of February 2002, Hackystat has undergone an architectural revision and some of these documents no longer reflect the current design of the system. In cases where the document largely reflects Hackystat 1 architectural concepts, the document is prefaced by [Hackystat 1]. 

  1. Insights from the Presto development project: Requirements from the IDE for automated data collection, Jitender Miglani
    This document summarizes the insights gathered from the design of Presto, a Kawa-specific approach to automated data collection and analysis.
     
  2. A roundtable discussion of Hackystat, Philip Johnson
    Provides some early issues and answers on the goals, purpose, and requirements for Hackystat.
     
  3. TaskHacker: Change management in Hackystat, Philip Johnson
    Provides an overview of the requirements for TaskHacker, a tool for supporting user stories, change requests, and bug reports in an XP and  Hackystat-compliant manner.
     
  4. [Hackystat 1] Validated idle time detection, Philip Johnson and Cam Moore
    Provides an overview of an approach to validating idle time detection and session boundaries in Hackystat.
     
  5. Defect collection and analysis in Hackystat, Philip Johnson
    Provides an overview of issues surrounding useful defect collection and analysis: what kinds of defects can be collected automatically, and what potential analyses could be performed on that data.
     
  6. Security and privacy in Hackystat, Philip Johnson and Shenyan Zhen
    Provides an overview of security and privacy considerations for the data collected and analyzed by Hackystat.
     
  7. [Hackystat 1] An annotated overview of Hackystat server extension, Philip Johnson
    Provides a 10 step, soup-to-nuts overview of implementing support for data collection and analysis on the server side.
     
  8. [Hackystat 1] Hackystat Public Server Release Process, Philip Johnson
    Explains the sequence of steps necessary to install a new binary release on bertha.
     
  9. Sensor data analysis in Hackystat, Philip Johnson
    What kinds of useful analyses can Hackystat provide based upon the types of data it can collect?
     
  10. Hackystat2 Design Proposal, Philip Johnson
    Ideas on how to improve the design of Hackystat.
     
  11. Analysis in Hackystat2, Philip Johnson
    A set of ideas on how to structure, design, and implement analysis mechanisms in the new architecture.
