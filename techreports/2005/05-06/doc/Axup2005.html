Comparison of Usability Evaluation Methods (UEMs)
Name Abbreviation Description Synopsis Advantages Disadvantages
Think Aloud User Testing Performance Testing Form of user testing where one user's interaction with a product is videotaped and analyzed to improve the product.
  • 1 participant
  • Uses scenario or tasks
  • Uses Novices or Domain experts
  • Users given instruction to "Think Aloud"
  • Minimal interaction between experimenter and user
  • Usually videotaped or recorded
  • Very close approximation to actual individual usage
  • Uses less users (less expensive)
  • Think aloud instruction rarely works well and is unnatural for the user
Constructive Interaction   Form of user testing where two users interaction with a product is videotaped and analyzed to improve the product.
  • 2 participants
  • One person "drives"
  • Uses scenario or tasks
  • Tasks are solved collaboratively
  • Users given instruction to "Think Aloud"
  • Uses Novices or Domain experts
  • Minimal interaction between experimenter and user
  • Usually videotaped or recorded
  • Produces an increased amount of verbal feedback concerning step by step, competitive problem solving methods and opinions.
  • May model real world behavior when we ask for help.
  • From anecdotal evidence it appears to find similar usability problems to the 1 participant model.
  • Approximates a collaborative problem solving process
  • Users more users (more expensive)
Focus Groups   Form of data gathering, usually used during the product conceptualization phase in the product cycle where potential users are asked for their opinions on a potential product.
  • A group of users is asked for their opinions about a product
  • Scenarios may be used
  • Demos may be used
  • Produces a large quantity of possible design choices and new features
  • May improve customer relations
  •  Is designed for a marketing tool
  • Is pseudo-scientific and is not a controlled environment
  • Does not test user's actual interaction with the software
  • Records what user's think they want, not necessarily what they would actually use. See Nielsen's "The user is always right, the user is not always right."
Expert Review ER People having sufficient experience or an advanced degree in a related discipline (HCI experts) critique a product separately or in groups to determine areas in need of improvement.
  • No guidelines
  • No Scenarios or tasks
  • Uses experts
  • Uses experts
  • May cover entire problem space
  • May be difficult to standardize or categorize rationale for design changes
Expert Walkthrough EW HCI experts use task scenarios to guide their analysis of the interface.
  • No guidelines
  • Uses scenarios or tasks
  • Uses experts
  • Uses experts
  • May focus on known problem areas
  • Inherent bias because of task selection
  • Does not cover entire problem space
Heuristic Evaluation HE HCI experts separately review an interface
and categorize and justify problems based on a short set of heuristics (rules of thumb).
  • Uses short guidelines
  • No scenarios or tasks
  • Uses experts
  • Uses experts
  • Gives multiple reviewers common rules to site for justification of reviews
  • Reasonably fast
  • The validity of Nielsen's guidelines have been questioned and alternative guidelines exist
Heuristic Walkthrough HW Similar to the above definition except task scenarios are used to guide the analysis.
  • Uses short guidelines
  • Uses scenarios or tasks
  • Uses experts
  • Uses experts
  • May focus on known problem areas
  • Inherent bias because of task selection
Guidelines G Guidelines range from several hundred items to roughly a thousand. A guidelines analysis judges whether an interface conforms to these rules.
  • Uses long guidelines
  • May be cheaper due to non-expert evaluators
  • Time intensive and tedious
  • Guidelines conflict and may be biased
Guidelines Walkthrough GW Uses the above mentioned guidelines but uses task scenarios to guide the analysis.
  • Uses long guidelines
  • Uses scenarios or tasks
  • May be cheaper due to non-expert evaluators
  • May focus on known problem areas
  • Time intensive and tedious
  • Guidelines conflict and may be biased
  • Inherent bias because of task selection
  • Does not cover entire problem space
Cognitive Walkthrough CW A method which fully utilizes task scenarios to stress the user's cognitive process and model, which guides the analysis.
  • Uses "information processing perspective" which puts the focus on the user's cognitive process and perception
  • Uses scenarios and tasks
  • Puts the focus on the user
  • May focus on known problem areas
  • Recognition of user goals
  • May be tedious
  • Tries to make the designer the user
  • Inherent bias because of task selection
  • Does not cover entire problem space


Taxonomy based in part on Gray + Salzman - Damaged Merchandise, 1998

Please note that UEM categorizations and names developed in an evolutionary fashion. Definitions have changed over time and studies that say they are using method A, may actually be using something closer to method B. This is often because the definition of A was open for interpretation at the time of testing, or a slight derivation of A was being attempted and had not yet been named. Hence, this is one of many possible taxonomies.
