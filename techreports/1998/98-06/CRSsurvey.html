Aberystwyth Software Engineering Group

http://www.aber.ac.uk/~dcswww/Public/Research/One_Pagers/SWENG.group.html

Pros:

  1. The page has all the information that you could want on one page. You are allowed to immediately branch to whatever information it is that you need.

Cons:

  1. Their logo, while colorful, is not clear. If you are going to go through the effort of loading something, it should be worth the wait time.
  2. There is a lot of information on the main page - this is not a problem because it is all in text and loads very quickly. The problem is that if your informational link was located at the end of the page, you would need to scroll down the entire page before you found it (actually, most of the links are located at the end of the document).

Site Summary:

This site seems to be more of a summary of the group - there was not too much that I could click around to. This seems to be common in quite a few of the sites that I have visited. If the site were more comprehensive, you could get a better feel for the group.

Application for CSDL:

I like the fact that the branching is very quick. If we can do the CSDL site in such a way that you branch immediately in a manner that is not too abrupt (see the Software Engineering Research Group at the University of Kaiserslautern , their branching system of the right-hand frame is really abrupt), it will help us with the problem of gearing the site to a wide variety of audiences.

BMRC - Berkeley Multimedia Research Center

http://www-plateau.cs.berkeley.edu/

Pros:

  1. My first reaction to the site is pretty good; it’s something I would expect from a Multimedia Research Center. Nice graphics, modern colors (a bit hard on the eyes, but that’s the trend now), and the formatting of the text blends nicely with the overall appearance.
  2. The site information page : http://www-plateau.cs.berkeley.edu/info/index.html. It explains about the site, what to expect, and because of the multimedia used here, it informs the user about the type of file formats in the site.
  3. The navigation through this site is great; subheadings throughout the pages are linked at the side for quick access, and this is done without the need for frames.

Cons:

  1. Although there is a text-version of the bottom image map, there is nothing to supplement the top image map. Although some of the links at the top image map are repeated at the bottom, there are some which are not, leaving the few who visit this site text-only to miss out on some sections.
  2. Some of the pages could have been split up a bit more. The "What is BMRC" page, http://www-plateau.cs.berkeley.edu/info/whatisbmrc.html, is very long, and although it features linked subheadings (see Pros #3), reading through all of the page is tiring.
  3. The "Media Editing Procedures" page, http://www-plateau.cs.berkeley.edu/info/procedures/, was quite a jump backwards. They could have at least made the background color the same. It gives a feeling of neglect for the site.

Site Summary:

I really like the design of this site—it’s modern looking, but it doesn’t take a long time to load the graphics. The designers made a great balance between the speed and looks for this site, something all sites should follow. Because of its great looks, it makes the group sound like something really exciting, instead of just another boring research group. Navigation around this site was fairly good, but some sections were very long, making it hard to look through. They do have a search engine, but it would have been nice if they offered other ways of categorizing their links. There is some documentation on the site itself, but only after you get there through the main page, which has none. I give this site an 8 out of 10.

Application for CSDL:

The first page seen by the user is very important--it makes the first impression. This page should look excellent (if not all of the pages), in order to draw the person in, and have them look further through the page.

Brown University Graphics Group Home Page

http://www.cs.brown.edu/research/graphics

Pros:

  1. A good search engine located directly on the first page, made for those who know the site, but don’t know where to find that particular document.
  2. Good use of graphics and theme throughout the pages. The sub-pages seem together with the main page because of the same theme used throughout the site.

Cons:

  1. The requirement for either Postscript or PDF readers. There should have been an HTML equivalent for the reports contained, since this is the WWW.
  2. At smaller resolutions, the main text of the page isn’t visible until scrolling down.
  3. There is a lack of information on the page. For instance, the Sketch page, http://www.cs.brown.edu/research/research/sketch/, only gives a brief overview of the Sketch concept, and related links, but no real information on the project.
  4. Bad server. While visiting the site, I was often unable to load sections because of the server having problems. I would visit a page, and when trying again, would be unable to view it again.

Site Summary:

I was not able to look through this site too thoroughly because of the server being ineffective at doing its job. The server went down twice in the five minutes I spent trying to view the site. If I was not specifically looking for this site, I would have given up by now. However, the site seems small, having only four main projects, but still there is not much information given on these sections. Many of the links to the major information require additional plug-ins, such as Adobe Acrobat, and although I tried viewing one, the server went down, and I was unable to do so. It seems that this site started out ambitiously, but peaked out before it was finished, as there is a very incomplete feel about this site. However, I can’t really judge too well, as I was not able to view the site in its entirety.

Application for CSDL:

This is a good example of using colors without using large graphics and of giving the effect of frames without really using frames.

  2. The site is explained on the main page, with link buttons on the side providing quick access to the sections contained.

Site Summary:

It took some looking through the site to determine what kind of laboratory this was. This site is very minimal, seeming to offering information only to those who know what they are looking for. Most of the sections only consist of the title and a listing of links; the one exception was the Research Facilities page (http://www.cclrc.ac.uk/Facilities/index.html) which had a nice description of their projects. However, upon visiting these links, I found many different styles of pages, which gave me the idea that these projects were totally unrelated to each other, and that this grouping together into a site was merely because of geographical reasons. This may be the case, but should be something avoided in the CSDL site.

A good explanation of the group should be present upon the first page. Without it, people who are actually looking for information related to CSDL may simply gloss over the site because of a misleading first page.

CeVis

http://www.mevis.de/english/CeVis

Pros:

  1. The main page explains how to navigate through the site, which is always good when using frames.
  2. This site has a great use of frames: it serves as a reminder of where you are, it provides quick navigation, and changes according to content when switching through sections.
  3. A great search engine, it allows for searching via many different criteria, and also gives a listing of key words to help the user.

Cons:

  1. The link to "Teacher Enh." in the navigation frame. It’s not apparent what "Enh." stands for, so it would’ve been better if they took two lines and actually wrote out "Enhancement". Furthermore, once you reach the site, the frames disappear.
  2. The links for the first CeVis Research page load another page with almost the same layout, but an exploded view of the section you chose. Basically, a waste of time, the exploded view of all of the research projects would have been fine, it could have been placed under the six main links if it was thought of as obtrusive.
  3. The bottom left hand frame annoyingly has a scroll bar; if the frame was made a little bigger, or if scrollbars were taken off the frame, this wouldn’t have happened.

Site Summary:

The CeVis site has that "European Dignity" feeling when first loaded. It isn’t exciting, but then it isn’t boring, just calmly dignified. The site could use more graphics, especially when the subject matter is complicated, such as the discussion on fractals. The use of frames is one of the better ones, as it not only unifies all of the pages with the logo at the top (although taking too much room at the top), but changes when going into a new section, making it very functional as a navigation tool. However, the annoying aspect of the bottom left frame having toolbars, and the fact that the entire page reload (as compared to just the frames reloading) when going to a new section cause a need for a little more tweaking. The sub-pages make you work a little harder than other sites; for instance, the staff page does not automatically link you to the member’s email addresses, you have to figure it out yourself using the "formula" for e-mail: FirstName.Lastname., and some sub-pages link only to an exploded version of itself instead of offering it as such in the beginning. The search engine (not available the second time I visited the site) was one of the best around, allowing users to search using a variety of criteria. A 7 out of 10, for good use of frames, a good search engine, but the need for a couple of tweaks.

Application for CSDL:

The search mechanism under Publications is something we should try to copy.

Computer Graphics at Stanford University

Site Summary:

For a graphics laboratory, this site is very low on pictures, not even using a background image. Most pages consist of just a bunch of pictures at the top, and a lot of text, which made reading hard and uninteresting. The page could have been made a lot nicer, and a lot more interesting, but because of using basically unorganized page layouts, it made this site very uninviting. About a 5 out of 10, leaning on the lower end.

Database Group @ University of Maryland

http://www.cs.umd.edu/areas/db/

Pros:

  1. The site allows you to look up information in a number of ways. For instance, to look up projects, you may either browse through the Projects section, or you may look through the People section to find it by author.
  2. The presentation of the DIMSUM project, http://www.cs.umd.edu/projects/dimsum/, was very interesting: although light in nature, there also was a serious side to it (see Recent Papers section), and it covered all the information fairly well. The graphics were a nice touch—not too extensive, but just enough to catch the eye.
  3. There is a lot of information about the group, I was impressed that they even included maps to their upcoming seminars.

Cons:

  1. Navigation around the site via the main page is difficult due to the undocumented one-word labels referring to the various major sections.
  2. In most cases, the simple line-art looks drab.
  3. The use of frames; in this case it wasn’t needed, and it doesn’t add to the site.
  4. It seems that after the main page, each page dedicated to a project is written by someone else. This gives a feeling of disunity for the research group.
  5. There could have been more internal links (see Pros #1). Although the Papers section was divided by author, these sub pages were not linked to the author in the Person section.

Site Summary:

Overall, I give this site a 6 out of 10. The graphics have a lot to work towards, but the text formatting is quite nice. The linking of sections together (see Pros #1) was a good idea, but more of this could be added. Unfortunately, the weakest part of this site are the sections on the projects itself. Most have only a short summary, with no links to anything else besides its parent page. The pages themselves look boring and thrown together, and are all different from each other. Because this is a research group, I think that more importance should have been put in the Projects section.

Application for CSDL:

Keep the site look consistent; especially because CSDL is a collaborative group.

Site Summary:

This site seems to be a site with the sole purpose of public information. It does a good job of presenting information on individual projects, but not on the group itself. It doesn't seem to be used by the group as a resource center, so that would limit the amount of information you could glean about the group itself. It is an information rich site, but not very user friendly. Navigation is not as easy as it could be and is rather boring. It reminds me of a hospital - sterile. There is not much color except for white and it seems to be geared for visitors. While there are elements that make it look like it is frequently updated ("new" signs), it doesn't give off that feeling.

Electronic Visualization Laboratory

http://www.evl.uic.edu/EVL/index.html

Pros:

  1. The page at http://www.evl.uic.edu/EVL/EVENTS/tq97.html is a great example of a good layout. It uses two columns, which shortens the line space, making it easier to read. It also includes graphics big enough to draw attention, but small enough to load quickly.
  2. The N.I.C.E. project, http://www.ice.eecs.uic.edu/~nice/NICE/intro.html, is a great example of effective use of color. The "virtual garden" theme of N.I.C.E goes well with the green and brown colors, relating to nature. This portion of the site also matches the mood of the project—since it was made for kids, it has a nice, playful attitude.
  3. The main page is set up rather well. At the top are quick links to sections, followed by a detailed description of each section. Finally, a search engine is available directly on the main page, for those who still don’t know which section to visit.
  4. Overall, the site uses a good selection of color. The yellow on black format makes it less strenuous to read, causing paragraphs to seem shorter than it would if it were black on white.

Cons:

  1. The pictures’ sizes are not written into the HTML code, so at times the page will jump after re-formatting itself when new pictures load.
  2. The top picture leads to a site I was unable to get to otherwise. It is not said where the picture link leads, but it is displayed prominently at the top, with all the link buttons pointing to it. Somewhat confusing.
  3. The logos for the sub-sections do not really match the main page, and furthermore, take too long to load. It’s also not worth loading, as it looks only like a bunch of dots.
  4. Some links should have been added. Because there are no links on the main page to the University of Illinois at Chicago, the lab does not seem part of the university. Also, email addresses are not linked.

Site Summary:

One of the better sites as regards consistency and layout. The use of dual columns in most of the pages was a nice touch; it allows for pictures to be inserted throughout the page without interrupting the flow of text, and also shortens the length of the lines of text, making it easier to read, especially when using 1024x768. The colors used were very calm, so reading the text was less strenuous than the harsh black on white that most other pages used. The sub pages’ design matched the main page, and this helped unite the entire site together, even though the tone of the writing changed (and in a good way) such as in the N.I.C.E. project page. Navigation was somewhat tricky, however, as most pages only linked back to the main page instead of the page that called it. Some links were needed as well, for instance, the email addresses for the members were not linked (in their individual pages). Also, the main page did not give enough information on the group, so unless you were looking specifically for the group, you would not bother looking further into the site unless you were interested in the graphic at the top. Overall, with a little cleaning, and maybe new graphics, this site can be great, and so I give it an 7 out of 10.

Application for CSDL:

Don't change font size or colors halfway through the page.

Application for CSDL:

SUMMARY

General Design:

The CSDL site should have a great looking first page, in order to draw the audience further into the site. Frames are not a necessary item in order for the page to look high tech. Frames can be synthesized by use of tables, which also offer the advantage of being able to be resized and centered in a page, lessening the amount of information lost if a browser were to be resized. Great information is useless unless the audience can find it. The site should explain the sections of itself so that people can navigate through the site easier. As for the esthetic qualities, the site should not go overboard with the graphics; the use of small icons to convey additional information (when appropriate) can be a nice touch.

Unity of the Site:

Especially because CSDL is a collaborative group, the individual pages in the site should look like they belong together. A color and font scheme should be used and kept throughout the site. If a large banner graphic is used on the main page, this can be shrunk and used in the child pages, perhaps as a link back to the main page. Lastly, a constant header and footer can keep the site together.

User Friendliness:

If a site is not easy to navigate through, then the user often will not bother. Branching out from the main page should be carefully balanced. It should not immediately end up to documents, but it also should not force users to look through five menus to find the information needed. If many pictures are going to be provided, thumbnails should be used. Furthermore, if graphical links are present, text alternative links should always be given. We also should have a standard navigation bar that lets you jump to any topic from any where within the site. Finally, be certain that all links given end up somewhere. A broken link gives a very bad impression.

Technology:

Going through the pages displayed certain technologies that may be used in the site. The CeVis search engine is one that could be copied, giving the user many ways in which to search for documents. However, counters seem to be an old, outdated technology. We could exclude that in our new site.
