Consumer

The consumer dimension parallels the producer dimension except that it characterizes what objects can be affected in the system by state information collected from the producers.

For example, a process object can be affected by the state of artifacts in the system. Systems supporting software engineering process modeling typically include rules such as: the testing phase cannot be entered until the state of the code artifact includes ``clean compile.''
